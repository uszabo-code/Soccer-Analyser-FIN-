# Soccer Analyser FIN

AI-powered soccer game analysis pipeline for Finnish youth club football. Tracks players across a full match, extracts performance metrics, and generates coaching reports — designed for fixed-angle elevated cameras (Veo Cam 2, Wisemen, Pixellot style).

---

## What It Does

Given a game video and a jersey number, the pipeline produces:

- **Distance covered** (total, sprint, walk, run zones)
- **Speed profile** (avg, max, time in each zone)
- **Heatmap** (field zone coverage)
- **Ball proximity** (how often the player was near the ball)
- **Off-ball movement** (spacing, positioning intelligence)
- **Team context** (player's role relative to teammates)
- **Coaching report** (generated by Claude AI)

---

## Pipeline Stages

```
Stage 1   Detection & Tracking     YOLOv8 (football_yolov8.pt) + BoT-SORT
Stage 1b  Track Merging            Post-process fragmented tracks by color + position
Stage 2   Player Identification    Jersey OCR via PaddleOCR
Stage 3   Feature Extraction       Distance, speed, heatmap, ball proximity
Stage 3b  Advanced Features        Spacing, off-ball movement, positional intelligence
Stage 4   LLM Analysis             Claude API — narrative coaching insights
Stage 5   Report                   Markdown report with charts
```

## Setup

### Requirements
- Python 3.9+
- macOS (MPS) or Linux (CUDA/CPU)
- ~4GB disk for models

```bash
git clone https://github.com/uszabo-code/Soccer-Analyser-FIN-.git
cd Soccer-Analyser-FIN-
./setup.sh
```

### Models

Download model weights separately (not included in repo due to file size):

| Model | Purpose | Source |
|---|---|---|
| `football_yolov8.pt` | Player + ball detection | [HuggingFace](https://huggingface.co/uisikdag/yolo-v8-football-players-detection) |

Place `.pt` files in the project root.

## Usage

```bash
# Analyse a full game for player #15
python analyze.py videos/game.mp4 --jersey 15

# Skip Claude analysis (faster, no API key needed)
python analyze.py videos/game.mp4 --jersey 15 --no-llm

# Resume from a specific stage (e.g. after Stage 1 is done)
python analyze.py videos/game.mp4 --jersey 15 --stage 3

# Override device and frame skip
python analyze.py videos/game.mp4 --jersey 15 --device cpu --skip 3
```

Output is saved to `output/` (configurable via `--output-dir`).

## Configuration

Key settings in `config.py`:

```python
YOLO_MODEL = "football_yolov8.pt"   # Detection model
FRAME_SKIP = 2                       # Process every 2nd frame (best continuity/speed tradeoff)
TRACKER_CONFIG = "botsort_soccer.yaml"  # BoT-SORT with GMC, no ReID
FIELD_WIDTH_METERS = 60.0            # Visible field width (U13–U15 camera crop)
SPRINT_SPEED_THRESHOLD = 4.0         # m/s
```

### Tracker Config (`botsort_soccer.yaml`)

Tuned for overhead fixed-angle cameras with pan/tilt:

```yaml
tracker_type: botsort
track_buffer: 90        # 3s gap tolerance — players re-enter after occlusion
match_thresh: 0.7       # Association similarity
gmc_method: sparseOptFlow  # Camera motion compensation
with_reid: False        # ReID disabled (OSNet not suited for overhead view)
```

## Camera Setup

Designed for **Finnish youth club cameras** — typically elevated fixed-angle cameras showing ~60m of field width. The pipeline is tuned for:

- Overhead angle (~15–30° elevation)
- Players at ~80–150px height in frame
- Camera pan/tilt (handled by BoT-SORT GMC)
- 1920×1080 @ 30fps input
